CSS

  • There is inline CSS styling and embedded/referenced CSS styling that is on a seperate css sheet whch is included in the project's head element under "list"
  • A margin indicates how much space we want outide an element (in between border and content)
  • A padding indicates how much space you want around the content inside of the element
  • A box-shadow is a shadow effect that is measured in X, Y format
  • Colors can be identified specifically by HEX code (link saved)
  • To create specific formatting instruction create a CSS class (i.e. .card { bleep blop bloop }) and then reference it by section (class="" id="")
  • .css

Git

  • git status: checks what branch we are currently working on
  • git checkout branch-name: switches to named branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -a: stages all edited files/revisions for commiting
  • git restore --staged'file-name': to unstage a file to commit
  • git commit -m 'comment/description': commits all changes added, effectively saves. Do not switch branches without commiting changes
