HTML

  • The head element contains information about the webpage
  • The code in the head element is not visible to webpage viewers once deployed
  • The head element contains meta and title information
  • links to referenced style sheets can also be placed in the head section and will be applied to entire html doc
  • meta information includes character set (UTF-8 seems standard) and other important informatino for the browser to display your information correctly.
  • The body element represents the visible content shown to the user
  • .html

CSS

  • There is inline CSS styling and embedded/referenced CSS styling that is on a seperate css sheet whch is included in the project's head element under "list"
  • A margin indicates how much space we want outide an element (in between border and content)
  • A padding indicates how much space you want around the content inside of the element
  • A box-shadow is a shadow effect that is measured in X, Y format
  • Colors can be identified specifically by HEX code (link saved)
  • To create specific formatting instruction create a CSS class (i.e. .card { bleep blop bloop }) and then reference it by section (class="" id="something that can help search engines find your article/image")
  • .css

Git

  • git status: checks what branch we are currently working on
  • git checkout branch-name: switches to named branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -a: stages all edited files/revisions for commiting
  • git restore --staged'file-name': to unstage a file to commit
  • git commit -m 'comment/description': commits all changes added, effectively saves. Do not switch branches without commiting changes
  • git pull origin main: pull any changes from main branch down to your local branch
  • git push origin branch-name: submits a pull request to main branch to review and merge with changes made. Next you should visit githib.com to Compare the files/merge if everything looks okay.
